# シンプルなWebノートアプリ計画

## 目的

まずはブラウザで使える，手書き中心のシンプルなノートアプリを作る。
最初はアカウントや同期を入れず，ノートの作成，手書き，検索，自動保存が気持ちよく動くことを優先する。
将来的にはPWA化し，インストールしてローカルでも作業できるようにする。

## 決定事項

- 最初の形: Webアプリ
- 初期配布: ブラウザで開ける静的アプリ
- 初期データ保存: ブラウザのローカル保存
- 基本入力: テキスト本文ではなく手書きキャンバス
- 将来対応: PWA，オフライン対応，IndexedDB，エクスポート/インポート
- 今回はiPadネイティブアプリやSwift実装は扱わない

## 想定する使い方

- PCやタブレットのブラウザで手書きメモを書く
- ノートを複数作る
- タイトルを付けて手書きページを整理する
- 最近更新したノートを探す
- タイトル検索で必要なノートを見つける
- ネット接続がない状態でも，将来的にはPWAとして作業を続けられる

## MVPで作る機能

- ノート一覧
- ノートの新規作成
- ノートのタイトル編集
- ペンでの手書き入力
- 消しゴム
- ペンの色と太さの変更
- ノート作成時の紙サイズ選択
- 背景の選択
- 紙面の拡大縮小
- undo / redo
- キャンバスのクリア
- ノートの削除
- 自動保存
- ローカル保存
- タイトル検索
- 更新日時の表示
- 空の状態でも迷わない画面

## 画面構成

### ノート一覧

- ノート一覧は独立したページとして表示する
- 新規作成ボタンをすぐ押せる場所に置く
- ノート名と更新日時を表示する
- 検索欄でタイトルを絞り込めるようにする

### 編集画面

- ノート一覧は表示せず，手書きに集中できる全画面構成にする
- 一覧へ戻るボタンを置く
- タイトル入力
- 手書きキャンバス
- ペン，消しゴム，色，線幅，背景，undo / redo
- 表示倍率の変更
- 保存状態の表示
- 削除ボタン
- 余計な装飾を抑え，書く領域を広くする

## 技術方針

最初は依存関係を増やさず，HTML / CSS / JavaScriptだけで作る。
小さく動くものを先に作り，必要になった段階でフレームワークやビルド環境を入れる。

- UI: HTML / CSS
- 動作: Vanilla JavaScript
- 初期保存: `localStorage`
- 描画: Canvas API
- 入力: Pointer Events
- 将来保存: `IndexedDB`
- 将来PWA: `manifest.webmanifest` と Service Workerを追加する
- 将来拡張: React / Vue / Svelteなどは必要になった段階で検討する

## 開発ステップ

### 1. Webアプリの最小版

- HTML / CSS / JavaScriptの基本構成を作る
- ノート一覧と手書き編集画面を作る
- 新規作成，編集，削除を実装する
- Canvasでペンと消しゴムを実装する
- `localStorage` に自動保存する
- タイトル検索を実装する

### 2. 使いやすさの改善

- モバイルとタブレットでレイアウトを調整する
- キーボード操作を整える
- 空状態と削除確認を改善する
- エクスポート/インポートを追加する

### 3. PWA化

- `manifest.webmanifest` を追加する
- Service Workerを追加する
- アプリの基本ファイルをキャッシュする
- オフライン時の動作を確認する

### 4. 保存層の強化

- `localStorage` から `IndexedDB` へ移行する
- ノート数が増えても重くなりにくくする
- データ移行の仕組みを用意する

## 最初のゴール

まずは「ブラウザで開いて，複数の手書きノートを作成，編集，検索，自動保存できる」ところを目標にする。

その後，PWA化とオフライン対応に進む。
