# シンプルなノートアプリ計画

## 目的

最初はiPadでの手書きを主な用途にした，軽いノートアプリを作る。最初は「書く」「消す」「ページを増やす」「保存する」が気持ちよく動くことを重視し，将来的にPCアプリやウェブアプリにも広げられる構成にする。

## 想定する使い方

- iPadとApple Pencilで手書きノートを取る
- 授業，会議，アイデアメモ，計算，図解に使う
- 複数ページのノートを作る
- あとからPCやブラウザでも閲覧，整理，軽い編集ができる
- ネット接続がなくても最低限使える

## 大事にする方針

- 手書きの遅延をできるだけ小さくする
- 起動してすぐ書ける
- 画面を広く使い，UIは必要最小限にする
- データを長く扱える形式で保存する
- 将来の同期やマルチデバイス対応を見据える
- 最初から機能を増やしすぎず，書き心地を優先する

## MVPで作る機能

最初の完成目標は，1人で日常的に使える最低限の手書きノートにする。

- ノート一覧
- ノートの新規作成，名前変更，削除
- ページの追加，削除，移動
- ペンでの手書き入力
- 消しゴム
- ペンの色と太さの変更
- undo / redo
- 自動保存
- ローカル保存
- 白紙，罫線，方眼のページ背景
- PDFまたは画像としての書き出し

## 後回しにする機能

最初から入れると複雑になりやすいので，MVP後に検討する。

- クラウド同期
- 共同編集
- 手書き文字認識
- 図形補正
- 音声録音
- タグ，全文検索
- PDF読み込みと注釈
- テンプレート共有
- アカウント機能

## 画面構成

### ノート一覧画面

- 最近使ったノートを上に表示する
- 新規ノート作成ボタンを置く
- ノート名，更新日時，ページ数を表示する
- 検索やフォルダはMVP後に追加する

### 編集画面

- 中央にページキャンバスを表示する
- 上部または左右に最小限のツールバーを置く
- ペン，消しゴム，色，太さ，undo，redo，ページ操作を使えるようにする
- iPadでは指でスクロール，Apple Pencilで描画できるようにする
- PCではマウス，トラックパッド，ペンタブレットでも操作できるようにする

## 技術方針

### アプリの形

最初はiPad向けアプリとして配布する。ただし，将来PCアプリやウェブアプリに広げる可能性があるため，ノートのデータ形式や描画モデルは特定環境に閉じすぎないようにする。

- 初期対象: iPad
- 初期配布: iPadアプリとして配布する
- 初期データ: ローカル保存のみ
- 将来展開: PCアプリやウェブアプリでも読めるデータ形式を意識する
- 同期: MVPでは扱わず，後から追加できる余地だけ残す

### 描画

手書きの快適さが最重要なので，描画部分はアプリの中心として設計する。

- iPadではApple Pencilに強い描画方法を優先する
- PencilKitを使うか，独自のストローク描画を実装するかをプロトタイプで比較する
- ストロークをビットマップではなくベクターデータとして保存する
- 入力点，筆圧，傾き，時刻などを必要に応じて記録する
- 表示用には高速に再描画できるキャッシュを持つ
- 将来ウェブ版を作る場合は，同じストロークデータをCanvasやWebGLで再描画できるようにする

### 保存形式

あとから他環境に移しやすいように，アプリ独自形式でも中身は読みやすくする。

- ノート単位でメタデータを持つ
- ページ単位で背景とストロークを保存する
- ストロークはJSONなどの構造化データで保存する
- 画像やPDFへの書き出しを別途用意する
- PDFの取り込みと注釈はMVPには含めず，後の機能として扱う

例:

```text
notebook
  id
  title
  createdAt
  updatedAt
  pages
    id
    background
    strokes
      id
      tool
      color
      width
      points
```

## 入力設計

- Apple Pencilでの描画を最優先にする
- 指はスクロール，ズーム，ページ移動などの操作を優先する
- 設定で「指でも描く」を選べるようにする
- 筆圧に対応している環境では線幅に反映する
- 将来のウェブ版ではPointer Eventsに変換できるよう，入力データは抽象化して扱う

## パフォーマンス方針

- 描画中は入力を止めない
- ストローク確定後に保存する
- 保存は自動で行い，ユーザーに意識させない
- ページ全体を毎回描き直さず，必要な範囲だけ更新する
- ページ数やストローク数が増えても重くなりにくい構造にする

## UI方針

- 画面上のボタンは少なくする
- よく使う操作は1タップで届く場所に置く
- 色や太さはすぐ切り替えられるようにする
- 書いている最中にツールバーが邪魔にならない配置にする
- ノートの管理よりも，書く体験を優先する

## 開発ステップ

### 1. プロトタイプ

- 1ページだけのキャンバスを作る
- ペンで線を描けるようにする
- 消しゴムを実装する
- undo / redoを実装する
- iPadで遅延や操作感を確認する

### 2. ノート機能

- ノート一覧を作る
- 複数ページを扱えるようにする
- ローカル保存を実装する
- 自動保存を実装する

### 3. 使いやすさの改善

- ペン色，太さ，背景を切り替えられるようにする
- ページ移動を快適にする
- PDFまたは画像として書き出せるようにする
- iPadでのUI配置を調整する

### 4. 将来対応の準備

- データ形式を安定させる
- インポート，エクスポートを整える
- 同期機能を追加できるように保存層を分離する
- PCアプリ化またはウェブアプリ化を検討する

## 技術候補

現時点では、以下の方向が候補になる。

- iPadアプリ: Swift / SwiftUI
- 手書き描画: PencilKit，または独自ストローク描画
- ローカル保存: アプリ内ファイル，SQLite，またはiOS標準の永続化機能
- 書き出し: 画像またはPDF
- 将来のウェブ版: React / Vue / Svelteなどを必要になった段階で検討する
- 将来のPCアプリ: ネイティブ実装，Tauri，Electronなどを必要になった段階で検討する
- 同期: MVPでは実装せず，あとからクラウドストレージやバックエンドAPIを追加する

## 決定事項

- 最初の対象タブレット: iPad
- 最初の提供形態: ブラウザではなく，iPadアプリとして配布する
- 初期データ方針: ローカル保存のみ
- PDF取り込み: MVPには含めず，後回しにする

## 最初のゴール

まずは「iPadアプリとして開いて，1冊のノートに複数ページの手書きメモをローカル保存できる」ところを目標にする。

この段階で書き心地が良ければ，その後にノート管理，書き出し，同期，PCアプリ化，ウェブアプリ化を順番に足していく。
