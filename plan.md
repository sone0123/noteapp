# シンプルなWebノートアプリ計画

## 目的

まずはブラウザで使える，手書き中心のシンプルなノートアプリを作る。
最初はアカウントや同期を入れず，ノートの作成，手書き，検索，自動保存が気持ちよく動くことを優先する。
PWAとしてインストールでき，基本ファイルはオフラインでも開けるようにする。

## 決定事項

- 最初の形: Webアプリ
- 初期配布: ブラウザで開ける静的アプリ
- 初期データ保存: IndexedDBによるブラウザのローカル保存
- 基本入力: テキスト本文ではなく手書きキャンバス
- 対応済み: PWA，基本ファイルのオフラインキャッシュ，PDFエクスポート
- 将来対応: インポート
- 今回はiPadネイティブアプリやSwift実装は扱わない

## 想定する使い方

- PCやタブレットのブラウザで手書きメモを書く
- ノートを複数作る
- タイトルを付けて手書きページを整理する
- 最近更新したノートを探す
- タイトル検索で必要なノートを見つける
- ネット接続がない状態でも，PWAとして基本的な作業を続けられる

- 紙面はA4，罫線固定
- 紙面の拡大縮小

- 選択ページの削除
- undo / redo

- ノート一覧からのノート削除
- 自動保存

## 画面構成

### ノート一覧

- ノート一覧は独立したページとして表示する
- 新規作成ボタンをすぐ押せる場所に置く
- ノート名と更新日時を表示する
- ノート一覧から削除できるようにする
- 検索欄でタイトルを絞り込めるようにする

### 編集画面

- ノート一覧は表示せず，手書きに集中できる全画面構成にする
- 一覧へ戻るボタンを置く
- タイトル入力
- 手書きキャンバス
- ペン，消しゴム，ストローク単位の消しゴム，3色の色，固定の細線，undo / redo
- 最後のページに描画がある時だけ空白ページを下に自動追加する
- 末尾の描画が消えた時は余分な空白ページを自動削除する
- 選択ページを削除する
- 表示倍率の変更
- 最終更新時刻とページ番号の表示
- 余計な装飾を抑え，書く領域を広くする

## 技術方針

最初は依存関係を増やさず，HTML / CSS / JavaScriptだけで作る。
小さく動くものを先に作り，必要になった段階でフレームワークやビルド環境を入れる。

- UI: HTML / CSS
- 動作: Vanilla JavaScript
- 初期保存: `IndexedDB`
- 描画: Canvas API
- 入力: Pointer Events
- PWA: `manifest.webmanifest` と Service Workerで基本ファイルをキャッシュする
- 将来拡張: React / Vue / Svelteなどは必要になった段階で検討する

## 開発ステップ

### 1. Webアプリの最小版

- HTML / CSS / JavaScriptの基本構成を作る
- ノート一覧と手書き編集画面を作る
- 新規作成，編集，削除を実装する
- Canvasでペンと消しゴムを実装する
- 通常の消しゴムに加え，ストローク単位で消せる消しゴムを実装する
- `IndexedDB` に自動保存する
- タイトル検索を実装する

### 2. 使いやすさの改善

- モバイルとタブレットでレイアウトを調整する
- キーボード操作を整える
- 空状態と削除確認を改善する
- エクスポート/インポートを追加する

### 3. PWA化

- `manifest.webmanifest` を追加する
- Service Workerを追加する
- アプリの基本ファイルをキャッシュする
- オフライン時の動作を確認する

### 4. 保存層の強化

- ノート数が増えても重くなりにくくする
- データ移行の仕組みを継続的に整える

## 最初のゴール

まずは「ブラウザで開いて，複数の手書きノートを作成，編集，検索，自動保存できる」ところを目標にする。

その後，保存層の強化とインポート機能に進む。
